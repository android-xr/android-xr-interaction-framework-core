# Contributing to the Android XR Interaction Framework

First off, thank you for using the Android XR Interaction Framework! We value your feedback and share your desire to make this library as robust and useful as possible.

Since the core of this library is **closed-source**, we handle contributions differently than traditional open-source projects. Please read this guide to understand how you can help.

## Before you begin

### Sign our Contributor License Agreement

Contributions to this project must be accompanied by a
[Contributor License Agreement](https://cla.developers.google.com/about) (CLA).
You (or your employer) retain the copyright to your contribution; this simply
gives us permission to use and redistribute your contributions as part of the
project.

If you or your current employer have already signed the Google CLA (even if it
was for a different project), you probably don't need to do it again.

Visit <https://cla.developers.google.com/> to see your current agreements or to
sign a new one.

### Review our community guidelines

This project follows
[Google's Open Source Community Guidelines](https://opensource.google/conduct/).


## How can I contribute?
Issue Tracker is a tool used internally at Google to track bugs and feature requests during product development. It is available outside of Google for use by external public and partner users who need to collaborate with Google teams on specific projects.

Public users will access Issue Tracker at https://issuetracker.google.com

- [User's guide](https://developers.google.com/issue-tracker)
- [FAQ](https://developers.google.com/issue-tracker/references/faq)
- [Terms of Service](https://developers.google.com/issue-tracker/#terms_of_service)

While the source code is not publicly available for direct modification, you can still contribute in several impactful ways:

### Reporting bugs
If you encounter a bug, please open an [Issue](https://issuetracker.google.com/issues/new?component=2131847&template=2323364) and include:
- **Version**: Which version of AXRIF are you using?
- **Environment**: Describe your OS (macOS/Windows/Linux), Architecture (arm64/x86_64), and Game Engine version.
- **Device**: Which Android XR-compatible device are you using?
- **Description**: A brief explanation of the issue you are encountering.
- **Reproducible Example**: A minimal code snippet or small project that demonstrates the issue.
- **Observed vs. Expected**: What happened, and what did you expect to happen?

### Feature requests
Have an idea to make the library better? We’d love to hear it!
To help us learn more, please open a [Feature Request](https://issuetracker.google.com/issues/new?component=2131847&template=2323492) and include:
- **Summary**: Provide a clear and concise description of the feature.
- **Problem Statement**: What is the problem this feature is solving? Describe the pain point or the "why" behind this request.
- **Proposed Solution**: Describe how you envision this feature working. Include any specific functionality or workflows.
- **User Impact**: Who is this feature for? How does it improve the experience for them?
- **Alternatives Considered**: Are there any workarounds or other ways to achieve this goal currently?
- **Additional Context / Screenshots**: Add any other context, mockups, or screenshots about the feature request here.
